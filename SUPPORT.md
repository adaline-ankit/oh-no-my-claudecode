# Support

## Questions

Use GitHub Discussions for setup questions, integration ideas, and broad design
questions:

https://github.com/adaline-ankit/oh-no-my-claudecode/discussions

## Bugs

Open a bug report when you have a reproducible failure. Include:

- the exact `onmc` command
- `onmc doctor` output when safe to share
- Python version, OS, and ONMC version
- whether the issue involves Claude Code hooks, MCP, Codex/`AGENTS.md`, sync, or mining

## Security

Use private vulnerability reporting for security issues. Do not post secrets, private
prompts, proprietary code, or full `.onmc/` databases in public issues.

## Contributor Fit

ONMC is intentionally focused on repo-native memory for coding agents. Contributions
are a good fit when they improve one of these paths:

- local setup and health checks
- provenance-aware memory extraction
- Claude Code hooks and MCP context
- Codex and cloud-agent restore flows
- portable `.agent-memory` workflows
- deterministic validation and debugging
